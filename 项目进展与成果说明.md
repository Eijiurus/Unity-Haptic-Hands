# 项目进展与成果说明

**文档性质**：基于当前仓库代码、`README.md` 与 Git 提交历史整理的阶段性说明，用于汇报毕业设计进展与可展示成果。  
**工程定位**：Unity 2022.3 LTS 上的「基于数据手套的虚拟手交互与触觉反馈」原型系统（本科毕业设计方向）。  
**生成日期**：2026-05-02  

---

## 一、项目目标（要解决什么问题）

在 Windows 上搭建一套可演示的闭环：**真实手指弯曲**驱动 **Rigged Hand 骨骼**，**手腕姿态与位移**来自 **BLE 惯性传感器**，场景中通过 **Trigger / 刚体** 完成触摸与抓取；触觉侧提供 **HID 板直连测试脚本**（及 README 中说明的 Arduino + DRV2605 扩展路径），便于验证马达与协议。

> 说明：`.cursorrules` 中曾提到 XR Interaction Toolkit；当前 `Packages/manifest.json` **未**引入 XRI，交互由自定义脚本与 Physics 完成，与主 `README.md` 描述一致。

## 三、成果展示（可按演示顺序对外说明）

### 1. 虚拟手与数据手套联动

- **成果**：五指弯曲实时映射到骨骼旋转；数据格式为手套 ASCII 行（分号结尾、11 个逗号分隔值，前 5 为弯曲通道），经 Python 去分号后以 UDP 整行送入 Unity。
- **对应实现**：`SensorBridge.py`、`GloveDataReceiver.cs`、`DataGloveHandDriver.cs`。

### 2. WitMotion 手腕跟踪

- **成果**：自动扫描名称含 **WT** 的 BLE 设备并连接；解析后用于手腕姿态，并支持更稳健的相对位置控制。
- **本轮进展**：`DataGloveHandDriver.cs` 已将位置部分从“依赖欧拉角补偿的自由积分”重构为 **慢速低通估计重力 + 启动静止标定 + 转动冻结窗口 + 弹簧回中** 的模型；并进一步把产品目标明确为**近场、短距离相对位移增强**（非远距离连续平移）：默认缩小 **`wristMaxOffset`**，可选 **`usePlanarPositionOnly`**（按轴向权重自动压掉最不稳定一轴，仅两轴参与位移）、**`positionRecenteringSpeed`**（运动过程中极慢拉回零以减轻长期累计），更适合答辩演示与近场抓取/碰触交互。
- **对应实现**：`WitMotionConnector.cs`、`Assets/Scripts/Bluetooth/` 下 `BleApi`、`BlueConnector`、`BlueScanner`，`Device/DeviceModel.cs`、`DevicesManager.cs`、`DataGloveHandDriver.cs`。

### 3. 场景内触摸与抓取

- **成果**：指尖球形 Trigger 检测可触摸物体；支持 **弯曲阈值捏取**、**拇指食指距离捏合**、**握拳抓取**、**G 键无手套捏取**；可选指尖物理碰撞球减轻穿模；可吸附刚体到指尖或手掌根。
- **对应实现**：`HandInteractionRig.cs`、`FingerTipRelay.cs`、`TouchableObject.cs`。

### 4. 编辑器与场景工作流

- **成果**：菜单 **Tools → Setup Rigged Hand Prefab** 一键从 FBX 生成左手 Prefab 并布置场景默认项；`HandSceneSetup` 使主摄像机跟随手部包围盒。
- **对应实现**：`Assets/Editor/RiggedHandPrefabSetup.cs`、`HandSceneSetup.cs`。

### 5. 控制台（ControlPanel）与物理 UI 原型

- **成果**：在 `SampleScene` 中可放置 **ControlPanel**（手前方约 30 cm）：灰色桌面、红/蓝两枚按压按钮（带透明触发体）、圆柱旋钮占位（含 `GrabZone` 触发球）、两盏由 `LampController` 驱动的灯泡（点光源 + 自发光材质）。菜单 **Tools → Setup Control Panel** 会生成层级与材质，并可将 **Button_01/02 → Lamp_01/02.Toggle** 持久化绑定（便于答辩一键还原）。
- **对应实现**：`Assets/Editor/ControlPanelSetup.cs`；`PressableButton.cs`、`ButtonTriggerZone.cs`、`LampController.cs`；材质目录 `Assets/Materials/ControlPanel/`。
- **与手套的关系**：控制台物体 **不需要** 挂 `GloveDataReceiver`（见下文附录）。手部交互依赖 **`FingerTip` Tag** + **`ButtonTriggerZone`** / 将来旋钮脚本读取指尖触发；`GloveDataReceiver` 仅负责把弯曲数据交给 **`DataGloveHandDriver`**（与可选的 **`HandInteractionRig`** 捏合逻辑）。旋钮旋转可在后续单独增加脚本（例如读 `HandInteractionRig` 或物理抓取），**不必**把每个控件都连到 `GloveDataReceiver`。

### 6. 触觉硬件侧验证

- **成果**：不依赖 Unity 即可对 HID 板发 **0xAA**（库振动）与 **0xBB**（遍历 DRV2605 效果 ID）命令，用于双路马达快速冒烟测试。
- **对应实现**：`test_hid.py`、`test_vibrate.py`。

## 附录：GloveDataReceiver 是什么？控制台 Button / Knob 要不要绑它？

**`GloveDataReceiver`（`Assets/Scripts/GloveDataReceiver.cs`）** 是 **手套数据入口**：从 **UDP**（`SensorBridge.py` 转发的串口数据）或 **键盘模拟**（无硬件调试）读取五指弯曲，解析为 **长度 5、0～1 归一化的 `FingerValues`**，供 **`DataGloveHandDriver`** 旋转手指骨骼；若场景里存在该组件，**`HandInteractionRig`** 也会在运行时自动找到它，用于 **捏合阈值 / 握拳抓取** 等基于弯曲的判断。

**不需要** 在 `ControlPanel` 下的 **Button、Knob** 上逐个绑定 `GloveDataReceiver`。推荐链路是：

1. 手套数据 → `GloveDataReceiver` → 骨骼弯曲；手腕 BLE → `DataGloveHandDriver` → 手整体位姿。  
2. 运行时 **`HandInteractionRig`** 在指尖骨骼下生成带 **`FingerTip` Tag** 的球形 Trigger。  
3. **按钮**：子物体 **`TriggerZone`** 上的 **`ButtonTriggerZone`** 检测 Tag，调用 **`PressableButton.OnFingerEnter/Exit`**，即可驱动下沉与 `UnityEvent`。  
4. **旋钮**：当前仅有 **`GrabZone`** 触发球占位；待你增加「旋钮逻辑」脚本后，同样可通过 **指尖 Trigger + Is Trigger 碰撞体** 或物理抓取实现旋转，**仍不必** 引用 `GloveDataReceiver`（除非你想用「某指弯曲角」直接映射旋钮角度，那是可选的另一套映射，再单独接数据即可）。

## 六、与「完全成品」之间的差距（诚实边界，便于答辩口径）

- **XR**：当前为桌面/非 XR 演示形态，未集成 XR Interaction Toolkit。  
- **触觉闭环**：Unity 内可通过 `TouchableObject` 的 `UnityEvent` 接串口/USB；与 HID 板的完整游戏内闭环需按固件协议对接（VID/PID 与报文可按实机修改）。  
- **Arduino + DRV2605**：README 列为可选路径；具体固件与串口协议以实际硬件为准。  
- **WT9011 位移边界**：当前方案强调**近场、短距离**“相对位移可演示、转动假位移更可控”，**不适合**当作远距离自由平移或绝对 3D 定位；除 `positionAxisWeight`、`gravityEstimateFilterSpeed`、`positionFreezeDuration`、`wristPositionSpring` 外，还可通过 `wristMaxOffset`、`usePlanarPositionOnly`、`positionRecenteringSpeed` 与 `wristVelocityDamping` 等结合佩戴方式与演示动作继续实测调优。  
- **资源占用**：`BlueScanner` / `BlueConnector` 与 `WitMotionConnector` 并存时注意 BLE 资源，README 已提示。
